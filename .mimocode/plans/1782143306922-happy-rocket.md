# Plan: Desglose de ahorros por método de pago

## Contexto
El usuario quiere ver el saldo de cada bolsillo/ahorro por separado (Nequi, Daviplata, Bancolombia, etc.) y poder sumar o restar de cada uno. Actualmente solo hay un total en `usuario.saldo_ahorros`.

---

## Solución: Agregar `saldo` a `metodos_pago`

### 1. Migración
**Archivo nuevo:** `supabase/migrations/00007_saldo_metodos.sql`

```sql
alter table public.metodos_pago add column if not exists saldo numeric not null default 0 check (saldo >= 0);
```

### 2. Dashboard — expanded view
**Archivo:** `components/dashboard/dashboard.tsx`

- Hacer clic en "Ahorros" abre un dialog con el desglose
- Cada método con saldo > 0 se muestra con nombre, tipo y saldo
- Botones +/- para agregar o retirar de cada uno

### 3. Actions para gestionar saldo
**Archivo:** `app/dashboard/actions.ts`

```ts
// Agregar saldo a un método
export async function agregarSaldoMetodo(metodoId: string, monto: number)

// Retirar saldo de un método
export async function retirarSaldoMetodo(metodoId: string, monto: number)
```

Ambas también actualizan `usuario.saldo_ahorros` (suma total).

### 4. Onboarding
**Archivo:** `app/onboarding/actions.ts`

Al guardar bolsillos, guardar saldo en `metodos_pago.saldo` en vez de solo en `usuarios.saldo_ahorros`.
