### Nuevo archivo: `supabase/migrations/00004_vehiculos.sql`

**Tabla `vehiculos`:**
- `id` (uuid PK), `user_id` (uuid FK→usuarios), `nombre` (text), `placa` (text nullable), `tipo` ('moto'|'carro'), `modelo` (text nullable), `anio` (integer nullable), `activo` (boolean), `created_at` (timestamptz)
- RLS con políticas SELECT/INSERT/UPDATE/DELETE para `auth.uid() = user_id`

**Alterar `documentos_activos`:**
- Agregar `vehiculo_id` (uuid FK→vehiculos, nullable, ON DELETE CASCADE)
- Agregar `precio_renovacion` (numeric ≥ 0)
- Mantener columna `vehiculo` (text) para retrocompatibilidad

**Modificar:** `supabase/full_schema.sql` con los mismos cambios

## Fase 5: Gestión de Vehículos (Múltiples)

### Nuevo: `app/vehiculos/page.tsx` (server component)
- Auth check → query `vehiculos` + `documentos_activos` → `<VehicleList />`

### Nuevo: `app/vehiculos/[id]/page.tsx` (server component)
- Query vehículo específico + documentos + gastos del vehículo → `<VehicleDetail />`

### Nuevo: `app/vehiculos/actions.ts` (server actions)
- `crearVehiculo(input)` → INSERT
- `actualizarVehiculo(id, input)` → UPDATE
- `eliminarVehiculo(id)` → DELETE (cascade docs)
- `renovarDocumento(docId, vehiculoId, nuevaFecha, precio)` → UPDATE doc + INSERT gasto en `transporte_mantenimiento`
- `crearDocumento(input)` → INSERT documento para vehículo

| `components/vehiculos/vehicle-form-dialog.tsx` | Dialog crear/editar: nombre, placa, tipo, modelo, año |
| `components/vehiculos/renewal-dialog.tsx` | Dialog renovar: fecha nueva, precio, confirmar → crea gasto |

### Modificar: `components/dashboard/alert-banner.tsx`
